# Terminal Playground Implementation Plan

## Overview
Add an interactive terminal console to the portfolio where users can execute commands to explore background information. Includes hover effects on skills for log messages.

## Requirements
- Terminal interface with command input and output display
- Commands: `help`, `projects --latest`, `socials`
- Skill hover logs: e.g., "[INFO] Initializing MongoDB connection... Success."
- Retro terminal styling with monospace font, dark background
- Responsive design

## Implementation Steps

### 1. HTML Structure
- Add new section `#terminal-playground` after projects
- Terminal container with:
  - Header: "Terminal Playground - API Console"
  - Output area (pre/code for formatted text)
  - Input field with prompt ">_"
  - Placeholder text

### 2. CSS Styling
- Terminal container: dark background, border, monospace font
- Output area: scrollable, green text on black
- Input: styled as terminal prompt
- Responsive: stack on mobile

### 3. JavaScript Functionality
- Command parser function
- Command handlers:
  - `help`: Display available commands
  - `projects --latest`: Show project details in formatted output
  - `socials`: Display GitHub/LinkedIn links in table format
- Input handling: Enter key to execute, clear input
- Output formatting: Use templates for consistent display

### 4. Skill Hover Logs
- Add mouseenter/mouseleave to `.stack-tag` elements
- Show log messages in terminal output on hover
- Different logs for different skills (e.g., MongoDB, Python, etc.)

### 5. Integration
- Position terminal section appropriately in layout
- Ensure theme compatibility (dark/light mode)
- Add navigation link if needed

## Technical Details
- Use `contenteditable` or input for command entry
- Store command history (optional)
- Format output with ANSI-like colors using CSS
- Test commands work correctly
- Ensure accessibility

## Testing
- Verify all commands work
- Test hover effects
- Check responsive behavior
- Validate in different browsers

## Timeline
- HTML/CSS: 1 hour
- JS core: 2 hours
- Hover logs: 1 hour
- Testing: 30 min</content>
